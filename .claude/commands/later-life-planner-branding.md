## Core Workflow

1. Clarify the brief before ideation.
   - Ask for long-term ambition, likely future products, target audience, tone, words to include or avoid, and hard constraints.
   - If the user has already supplied enough context or explicitly wants options now, proceed and state the assumptions instead of stalling.
   - Treat any rejected naming pattern as a hard rule for the rest of the session.
2. Separate the naming layers.
   - Treat legal company name, trading name, product name, and domain as separate decisions.
   - Prefer a distinct legal entity name when the public-facing brand risks a Companies House conflict.
3. Generate names in families.
   - Group options into conceptual, aspirational, journey/progress, fintech-style coined, and premium/credible names when useful.
   - Default to one- or two-word public-facing names.
   - Avoid three-part names, command-style constructions, and mixed-metaphor names unless the user explicitly asks for them.
   - For each name, explain why it works, the tone, and a plausible domain pattern.
4. Reduce the list hard.
   - Select the strongest shortlist rather than leaving the user with a flat brainstorm.
   - Distinguish marketing strength from registration risk when they differ.
5. Verify unstable facts from official sources.
   - For company-name availability or "too like" risk, use GOV.UK and Companies House.
   - For trademarks, use the official UK IPO search if the user asks.
   - For domains, verify with live DNS or WHOIS.
6. Make the recommendation concrete.
   - End with a recommended naming structure and the next checks needed.

## Brand Guardrails For This Project

- Keep the tone modern fintech, not pension-provider or insurer.
- Favor optimism, clarity, confidence, progress, independence, and reward after effort.
- Avoid names that feel old-fashioned, bureaucratic, gimmicky, or clinical.
- The first product is later-life financial planning, but the company name should stay broad enough for multiple future financial products.
- Prefer short, pronounceable names. Two-word names are fine if they are distinctive and easy to say.
- Prefer one- or two-word names that can stand alone as a brand.
- Avoid three-part names and names that read like a slogan or campaign line.
- Avoid mixing multiple conceptual themes in one name. Each candidate should live in one clear metaphor or concept family.
- If a name needs a colon or explanatory suffix to make sense, it is probably weak.

## UK Naming Research Rules

- Use concrete dates when discussing current availability.
- Treat these as separate checks:
  - company-name availability
  - trademark clearance
  - domain availability
- Use official sources first:
  - `https://www.gov.uk/limited-company-formation/choose-company-name`
  - `https://www.gov.uk/government/publications/incorporation-and-names/incorporation-and-names`
  - `https://find-and-update.company-information.service.gov.uk/company-name-availability`
  - `https://www.gov.uk/search-for-trademark`
- If the UK IPO search cannot be queried from the current environment, say so directly and do not imply trademark clearance.
- "No exact company name matches found" is not the same as final registrability.
- If an existing active company already owns the distinctive core words, treat that as a practical brand risk even if a longer variation passes the exact-match checker.

## Recommended Response Shapes

- Discovery: ask 5 to 8 focused questions.
- Ideation: give grouped options with rationale, not a raw list.
- Shortlist: rank the best 5 to 10 names and explain the tradeoffs.
- Risk check: answer in four parts:
  - company-name position
  - trademark position
  - domain position
  - practical confusion risk
